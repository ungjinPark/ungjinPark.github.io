Set Your First TreeView

Let's get install and set your first Treeview in HTML Docs.

You can import the Treeview material-pattern script by using the script tag such as the below line.

                    
<script src="treeview.js"></script>

Is the End! It's very simple! You don't need to work the script handling additional basically!

You should just import the script and write the markup on your webpage according to the specified pattern.

Script handling will be needed just for the custom click event only!.

Markup Structure

You can markup HTML according to the below code structrue to create the tree list.

                    
<ul role="tree" aria-label="tree example">
  <li><button role="treeitem">Example1</button><li>
  <li><button role="treeitem">Example2</button><li>
  <li><button role="treeitem">Example3</button>
    <ul>
      <li><button role="treeitem">Example1</button><li>
      <li><button role="treeitem">Example1</button><li>
      <li><button role="treeitem">Example1</button><li>
      <li><button role="treeitem">Example1</button><li>
    </ul>
  </li>
  <li><button role="treeitem">Example1</button></li>
</ul>

The Developers Must create a Treeview according to the above structrue.

Above code structure is the promised rule in pattern library.

But the tree-item element might be allowed to changing the base element to use. for example, you don't need to use the Button tag likes the above structure example necessarily. You might use an A(anchor) Tag,

Markup HTML to set default state and default event

Basically, you don't need to set state-related attributes in HTML.

But, I know some developers may want the optional function that sets an initial default state of the elements in a particular process of the page.

So, the markup pattern of the treeview.js is supports the setting method that sets the initial default value on HTML like the below code. 


<ul role="tree" aria-label="select directory">
  <li><button role="treeitem" aria-expanded="true">This PC</button>
    <ul>
      <li><button role="treeitem" aria-controls="C_Directory_Panel" class="browsed-pointer" aria-current="page">C:</button></li>
      <li><button role="treeitem" aria-controls="D_Directory_Panel">D:</button></li>
      <li><button role="treeitem" aria-controls="E_Directory_Panel">E:</button></li>
      <li><button role="treeitem" aria-controls="F_Directory_Panel">F:</button></li>
    </ul>
  </li>
  <li><button role="treeitem">Desktop</button></li>
  <li><button role="treeitem">NetWork-Drive</button>
    <ul>
      <li><button role="treeitem">Document</button></li>
      <li><button role="treeitem">Picture</button></li>
      <li><button role="treeitem">Video</button></li>
      <li><button role="treeitem">Music</button></li>
    </ul>
  </li>
  <li><button role="treeitem">Example1</button></li>
</ul>
<div class="panel"><!--for example-->
  <div id="C_Directory_Panel">...</div>
  <div id="D_Directory_Panel">...</div>
  <div id="E_Directory_Panel">...</div>
  <div id="F_Directory_Panel">...</div>
</div>

References

aria-expanded="boolean"(state)

Attribute target: tree-items that has sub-tree items

You can make the tree item set to expanded that has sub-tree-items when the document is ready.

class="browsed-pointer"(state):

Attribute target: One of the tree-items that users-visible currently

You can set default focus of the Treeview items

You can make users will be beginning navigate a specific item that was set to this class name.

DO NOT use to a hidden sub tree-item.

aria-controls="panel-content-id"(set default function)

Attribute target: the tree-item you want links to the readable panel content.

You can set the readable content container. It'll be set the activate-event automatically.

You can't use the custom event with the aria-controls at the same time..

aria-current="page"

Attribute target: the tree-item you want that will be activated panel content initially.

You can make the panel content of a specific tree-item activate
