the basic files of Treeview.js package

You must import the basic script files of the next file list at your document.

  1. treeview.js[main file]:Treeview object sources are in here
  2. treeview.config.js[running auto-creation of the treeview]:It makes the main script file run the creation of the Treeviews by recognizing the Treeview.js pattern on your document. And you can set the message text of tree hint announcement here

Set Your First TreeView

Let's get install and set your first Treeview in HTML Docs.

You can import the Treeview material-pattern script by using the script tag such as the below line.

                    
<script src="treeview.js"></script>
<script src="treeview.config.js"></script>

Is the End! It's very simple! You don't need to work the script handling additional basically!

You should just import the script and write the markup on your webpage according to the specified pattern.

Markup Structure

You can markup HTML according to the below code structrue to create the tree list.

                    
<ul role="tree" aria-label="tree example">
  <li><button role="treeitem">Example1</button><li>
  <li><button role="treeitem">Example2</button><li>
  <li><button role="treeitem">Example3</button>
    <ul>
      <li><button role="treeitem">Example1</button><li>
      <li><button role="treeitem">Example1</button><li>
      <li><button role="treeitem">Example1</button><li>
      <li><button role="treeitem">Example1</button><li>
    </ul>
  </li>
  <li><button role="treeitem">Example1</button></li>
</ul>

The Developers Must create a Treeview according to the above structrue.

Above code structure is the promised rule in pattern library.

But the tree-item element might be allowed to changing the base element to use. for example, you don't need to use the Button tag likes the above structure example necessarily. You might use an A(anchor) Tag,

References

aria-expanded="boolean"(state)

Attribute target: tree-items that has sub-tree items

You can make the tree item set to expanded that has sub-tree-items when the document is ready.

class="browsed-pointer"(state):

Attribute target: One of the tree-items that users-visible currently

You can set default focus of the Treeview items

You can make users will be beginning navigate a specific item that was set to this class name.

DO NOT use to a hidden sub tree-item.

aria-controls="panel-content-id"(set default function)

Attribute target: the tree-item you want links to the readable panel content.

You can set the readable content container. It'll be set the activate-event automatically.

You can't use the custom event with the aria-controls at the same time..

aria-current="page"

Attribute target: the tree-item you want that will be activated panel content initially.

You can make the specific tree-item activated when users opened your page

Library References: Introduction

This is for explaining how to programmatically control the Treeview elements provided by Treeview.js.

The treeview object is configured into the next three major objects.

  1. Tree (class: Tree)
  2. TreeItem (class: TreeItemElement)
  3. SubTree (class: SubTreeElement)

Focus Management

This reference page is for addressing all properties that manage the focus of tree-items.

Setters

Tree.setBrowsedPointer

    
        TreeInstance.setBrowsePointer = index:number (Seetter)

You can set a tree item of the specific index to be focusable in the Treeview. keyboard navigation events are running powered by this setter property.

Tree.setBrowsePointer


TreeInstance.moveBrowsePointerAndFocus=number (Setter)

This property sets the focusable tree-item and then send focus directly, you can use it the same as TreeInstance.setBrowsePointer

Getters

Tree.getBrowsedPointerElement

    
        TreeInstance.getBrowsedPointerElement:Element (Getter)

This property returns the currently focusable element.

Handling default event or activate custom event of the tree-items

Default Event Method

Treeitem.showPanel():void

TreeItemInstance.showPanel();

available method for tree-items only that has aria-controls

This property makes the content of this tree-item object activated. If treeitem has the custom click event, it won't working.

Native Method

Element.click()


const treeitem = document.querySelectorAll('li[role="none"]>[role="treeitem"]')
treeitem[1].click();

You can also simply make the content activated by using the click event method of DOM Element.

Managing Expand and Collapse State

This page is for explaining property that get and set the aria-expanded state

Getter And Setter

Treeitem.state_expand=boolean (Setter)

TreeItemInstance.state_expand = boolean

available method for tree-items only that has sub-tree items.

This property makes the sub-tree of this tree-item object expanded or collapsed.

If you want to make the sub-tree expanded, put the "true" boolean value here. Or not, put the "false" boolean value here.

Treeitem.state_expand=boolean (Setter)

TreeItemInstance.state_expand:boolean //>> true

This property returns a boolean type value. If aria-expanded Attribute is null or 'false', it will return a false boolean. and if is not null or false, will return true boolean.

Handling Click Event For Treeitems

You can use your custom event handler function for the Treeitems as the below line.


var customClick = Tree.collection[0].getAllTreeItems[0].setActivateEvent = handler;

But, remember! You can't set the custom click if you set the default function that displays the content.

Setting the Hint Messages for the Screen Readers

You can set the interaction hint messages for the screen readers by using next static properties in treeview.config.js. 


if(document.documentElement.lang === 'en'){
    Tree.setInteractionTipMessage = "You're in the Treeview, navigate with the Arrow keys.";
    Tree.ContentAvailableText = "Content's Available. Press Enter key to display at the panel area and then press the arrow down to leave the tree-view area and try to read the panel";
    Tree.setContentLoadedMessage = "Content loaded";
}
if(document.documentElement.lang === 'ko'){
    Tree.setInteractionTipMessage = "현재 트리뷰 항목에 있습니다. 방향키로 탐색하세요";
    Tree.setContentAvailableMessage = "읽을 수 있는 컨텐츠가 있습니다. 엔터 키를 눌러 패널에 내용을 표시하고, 탭 키로 트리뷰에서 벗어나 내용을 읽을 수 있습니다.";
    Tree.setContentLoadedMessage = "콘텐츠 로드됨";
}

About Treeview Creation programmatically

You can create the Treeview programmatically by using the methods of the below codes. 


const tree = document.createElement('ul');
const install_target = document.querySelector('.my-tree-container');
tree.setAttribute('role','tree');
tree.innerHTML=`
<ul role="tree" aria-label="tree example">
  <li><button role="treeitem">Example1</button><li>
  <li><button role="treeitem">Example2</button><li>
  <li><button role="treeitem">Example3</button>
    <ul>
      <li><button role="treeitem">Example1</button><li>
      <li><button role="treeitem">Example1</button><li>
      <li><button role="treeitem">Example1</button><li>
      <li><button role="treeitem">Example1</button><li>
    </ul>
  </li>
  <li><button role="treeitem">Example1</button></li>
</ul>    
`;
install_target.appendChild(tree);
Tree.collection[0].addTreeview(tree);
